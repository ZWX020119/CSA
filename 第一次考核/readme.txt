本次考核没有使用别的地方写好的框架，由html+css+js完成，项
目中使用到阿里iconfont对天气图标进行设计，使用到ps工具，和echarts.js画图。
实验总共调用四个接口
分别是：
实时天气接口：https://v0.yiketianqi.com/api?version=v61&appid=88638596&appsecret=qL7mw4JP
七日天气接口：https://v0.yiketianqi.com/api/worldchina?appid=88638596&appsecret=qL7mw4JP
24小时天气接口：https://v0.yiketianqi.com/api?version=v9&appid=88638596&appsecret=qL7mw4JP
生活指数接口：https://www.tianqiapi.com/life/lifepro?appid=88638596&appsecret=qL7mw4JP
总体分为两个主要页面：主页面 和  搜索页面
一丶主页面：分为顶部实时天气板块，今明两天天气板块，逐小时天气板块，七天天气板块，以及生活指数板块
1.顶部实时天气板块:最顶端中央部分显示地区，同时设置点击事件，实现与搜索界面交互的功能。左侧小板块显示空气质量，
中间部分显示当前气温，天气，以及风向。底部文字提示当前温度舒适度。
该板块使用到的js技术：
调取实时天气接口，根据获取到的数据进行板块设计。其中背景图片根据当前气候（晴，阴，雨等）对应切换。同时显示获取
到的温度，风向，气候。下方温度舒适度根据当前温度切换，（现在温度比较适宜~，现在温度比较凉爽~，天气寒冷注意保暖~ 等）
2.今明两天天气板块：调用七日天气接口，将今明两天的天气数据进行展示。增加事件，根据当前气候改变对应天气icon。
3.逐小时天气板块：该板块需要设计局部滚动。
采用方法：将外层父盒子css样式设为
给定高度和宽度，同时设置
white-space: nowrap;
 overflow-x: auto;
子盒子设置每小时一个小盒子，css样式设置为
display: inline-block;
通过该方法实现该板块滚动。
该板块使用到的js技术：
调用逐小时天气接口，获取时间，气候，温度，根据温度改变气候icon；
4.七天天气板块
同样采用局部滚动，方法同逐小时天气板块。
温度曲线图使用能够echarts.js绘制折线图，通过设置smooth属性设为平滑曲线，同时隐藏x，y轴，将数值显示在结点上方。
调用七天天气接口，获取日期，气候，风向，风速数据，显示在该板块。
5.生活指数模块
设置两个div盒子，每个里面有八个不同生活指数。
该板块使用到的js技术：
通过设置点击事件，将两个盒子交换显示。调取生活指数接口，获取生活指数数据。同时给每个生活指数增加点击事件，点击后显示详细内容方框提示。
二丶  搜索页面
由输入框，历史搜索，热门城市，搜索按钮组成。
获取城市数据方法：
1.通过js，获取搜索框内容，调取当前输入城市的天气。
2.点击热门城市获取当前热门城市天气。
历史搜索
通过js将搜索框和热门城市所查询的城市，每次生成一个新的li，插入到历史搜索板块内。
